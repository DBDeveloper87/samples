In general practice, a banner is created primarily from the CSS side of web development by way of the background-image property in a CSS class. For example:

However, one issue with this is primarily with the Apple products like iOS and macOS, when Smart Invert is enabled, the background image itself is inverted but any HTML images defined are not as shown in the example above.

A Workaround

There is, luckily a workaround for this. It involves stacking layers onto the page by way of the z-index (the third dimension)

We will create three classes, .banner, .backplane, and .frontplane. The .banner class is the “wrapper” that holds the two layers together, like a picture frame. The .backplane class is the photo in the frame (the night sky). Finally, .frontplane class is the glass of the photo frame that holds the actual content (the image of Mars and the “hello World!” text.

Detecting the Use of Invert Colors

Websites can detect that the user is using Invert Colors. In the examples above, when invert colors is not in use, the text is white. When Invert Colors is in use, the text appears white but is actually black because it is displaying the inverse of black.

To achieve this, we will use Media Queries, an advanced technique in CSS3 that is normally used for responsive design to detect the size of the screen and adjust accordingly.

Detecting Dark Mode

On many platforms today, the system now has Light or Dark Mode. Websites can detect which is being used and switch over to a different set of styles related to that color scheme. Again, this is another advanced technique in CSS derived from Media Queries.

One benefit of designing for Dark Mode is that unlike Invert Colors, the display is actually consistent across browsers that support this media feature.
